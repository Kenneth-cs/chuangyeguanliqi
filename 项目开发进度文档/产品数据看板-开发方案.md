# 产品数据看板 — 完整开发方案 v2.0

> 目标：在左侧菜单栏新增「产品数据」入口，支持外部产品（iOS App 等）通过 API Key 上报用户行为事件，并在独立看板中按产品维度查看 DAU、留存率、事件统计等运营数据。
>
> 首个接入项目：**钱小满（Money Full）iOS App**

---

## 一、整体架构

```
iOS App
  └─ 触发埋点事件
       └─ POST /api/events（携带 API Key）
            └─ 存入 AppEvent 表
                 └─ 产品数据看板页面查询展示
                      ├─ DAU 日趋势
                      ├─ 事件统计（按中文名聚合）
                      └─ 留存率（D1 / D7 / D30）
```

---

## 二、数据库新增表

### 2.1 AppEvent — 事件流表

```prisma
model AppEvent {
  id          String   @id @default(cuid())
  projectId   String
  deviceId    String            // 设备 ID，代表一个匿名用户
  eventId     String            // 事件标识符，如 record_submit_success
  eventName   String            // 中文名称，如「记账成功」，由上报方传入
  params      Json?             // 事件附加参数（金额区间、分类等）
  occurredAt  DateTime          // 事件发生时间（客户端时间）
  createdAt   DateTime @default(now())

  project     Project  @relation(fields: [projectId], references: [id], onDelete: Cascade)

  @@index([projectId, occurredAt])
  @@index([projectId, eventId, occurredAt])
  @@index([projectId, deviceId, occurredAt])
}
```

### 2.2 ApiKey — API Key 管理表

```prisma
model ApiKey {
  id          String    @id @default(cuid())
  userId      String
  key         String    @unique   // sk_live_xxxxx（生成后只展示一次）
  label       String              // 备注名，如「钱小满 iOS」
  createdAt   DateTime  @default(now())
  lastUsedAt  DateTime?           // 最后使用时间

  user        User      @relation(fields: [userId], references: [id], onDelete: Cascade)
}
```

## 三、API 接口设计

### 3.1 事件上报

```
POST /api/events
Authorization: Bearer sk_live_xxxxx
```

请求体：
```json
{
  "projectId": "xxx",
  "deviceId": "device-uuid",
  "eventId": "record_submit_success",
  "eventName": "记账成功",
  "params": {
    "type": "expense",
    "category": "餐饮",
    "amount_level": "level_1_under100"
  },
  "occurredAt": "2026-04-20T14:32:11Z"
}
```

### 3.2 DAU 查询

```
GET /api/analytics/dau?projectId=xxx&days=30
```

返回：
```json
[
  { "date": "2026-04-20", "dau": 340 },
  { "date": "2026-04-19", "dau": 312 }
]
```

### 3.3 事件统计查询

```
GET /api/analytics/events?projectId=xxx&days=7
```

返回：
```json
[
  { "eventId": "record_submit_success", "eventName": "记账成功", "count": 1240, "deviceCount": 320 },
  { "eventId": "project_create_success", "eventName": "成功创建项目", "count": 87, "deviceCount": 65 }
]
```

### 3.4 留存率查询

```
GET /api/analytics/retention?projectId=xxx
```

返回：
```json
[
  { "cohortDate": "2026-04-01", "newUsers": 100, "d1": 40, "d7": 15, "d30": 8 },
  { "cohortDate": "2026-04-02", "newUsers": 85, "d1": 38, "d7": 12, "d30": null }
]
```

### 3.5 API Key 管理

```
GET    /api/apikeys        — 查询当前用户的所有 Key
POST   /api/apikeys        — 生成新 Key（仅此时返回明文）
DELETE /api/apikeys/[id]   — 删除 Key
```

## 六、钱小满 iOS App 上报规范

### 6.1 事件 ID 与中文名对照表

| eventId | eventName（上报时携带） |
|---------|----------------------|
| `onboarding_view_page` | 浏览引导页 |
| `onboarding_click_skip` | 点击跳过引导 |
| `onboarding_complete` | 完成引导 |
| `record_click_add` | 点击记一笔入口 |
| `record_submit_success` | 记账成功 |
| `record_cancel` | 取消记账 |
| `project_click_new` | 点击新建项目 |
| `project_create_success` | 成功创建项目 |
| `project_view_detail` | 查看项目详情 |
| `analytics_view_page` | 浏览统计页 |
| `analytics_click_report` | 点击深度报告 |
| `profile_change_theme` | 切换外观主题 |

> **规则：** `eventName` 由 iOS 端上报时携带，系统按实际上报内容展示，新增事件无需后端配置，自动出现在看板。

### 6.2 Swift 上报示例

```swift
func trackEvent(eventId: String, eventName: String, params: [String: Any]? = nil) {
    let body: [String: Any] = [
        "projectId": "YOUR_PROJECT_ID",
        "deviceId": UIDevice.current.identifierForVendor?.uuidString ?? "unknown",
        "eventId": eventId,
        "eventName": eventName,
        "params": params ?? [:],
        "occurredAt": ISO8601DateFormatter().string(from: Date())
    ]
    
    var request = URLRequest(url: URL(string: "https://your-domain.com/api/events")!)
    request.httpMethod = "POST"
    request.setValue("application/json", forHTTPHeaderField: "Content-Type")
    request.setValue("Bearer sk_live_xxxxx", forHTTPHeaderField: "Authorization")
    request.httpBody = try? JSONSerialization.data(withJSONObject: body)
    
    URLSession.shared.dataTask(with: request).resume()
}

// 使用示例
trackEvent(
    eventId: "record_submit_success",
    eventName: "记账成功",
    params: ["type": "expense", "category": "餐饮", "amount_level": "level_1_under100"]
)
```

## 七、隐私合规说明

- `deviceId` 使用 iOS `identifierForVendor`（IDFV），不跨 App，符合 App Store 规范
- 不上报具体金额，只上报金额区间（`amount_level`）
- 不上报用户备注内容
- 仅在用户同意隐私协议后初始化上报功能
